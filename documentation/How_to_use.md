2. Вибір вектора атаки
Визначтеся, який канал зв'язку ви хочете тестувати:
- Telegram-модуль: Використовує Telegram Bot API та клієнтський протокол (Telethon).
- Dropbox-модуль: Використовує Dropbox API через .dat та .ack файли-маркери.

3. Підготовка машин (Attacker & Victim)
Для повноцінного тестування вам знадобляться два пристрої (або дві віртуальні машини):
- Server (Машина отримувача): Тут буде запущено скрипт server_basic.py або dropbox_server.py.
- Client (Машина відправника): Тут буде працювати client_basic.py або dropbox_client.py.

4. Налаштування конфігурації (.env) та OPSEC
Це найважливіший етап. У кореневій папці проєкту (або поруч із робочими скриптами) створіть файл з назвою .env і заповніть його за цим шаблоном:

API_ID=
API_HASH=
BOT_USERNAME=
TARGET_FOLDER=target_folder
PUBLIC_KEY_URL=
TELEGRAM_TOKEN=
BWS_ACCESS_TOKEN=
SECRET_ID=
DROPBOX_APP_KEY=
DROPBOX_APP_SECRET=
DROPBOX_REFRESH_TOKEN=

Де взяти ці дані?
Telegram:
Створіть бота через @BotFather. Ви отримаєте TELEGRAM_TOKEN та ім'я бота (BOT_USERNAME, наприклад @my_stealth_bot).
Авторизуйтесь на my.telegram.org, створіть додаток і отримайте API_ID (число) та API_HASH (рядок).

Dropbox:
Зайдіть у Dropbox App Console.
Створіть новий App, згенеруйте DROPBOX_APP_KEY, DROPBOX_APP_SECRET та налаштуйте постійний DROPBOX_REFRESH_TOKEN (з правами files.content.read та files.content.write).

Bitwarden Secrets Manager (BWS):
Створіть Machine Account, дайте йому доступ до вашого проєкту і згенеруйте токен (BWS_ACCESS_TOKEN).
Створіть секрет для збереження приватного ключа та скопіюйте його ID (SECRET_ID).

Генерація ключів шифрування (OPSEC-протокол):
Щоб уникнути зберігання ключів на жорсткому диску, виконайте наступне:

Запустіть скрипт generate_key.py. Він створить два файли: private_key.pem та public_key.pem.
Відкрийте private_key.pem, скопіюйте весь текст і вставте його як значення вашого секрету в Bitwarden.
Відкрийте public_key.pem, скопіюйте текст і створіть новий файл на GitHub Gist. Натисніть кнопку Raw, скопіюйте посилання з браузера і вставте його у .env як PUBLIC_KEY_URL.

КРИТИЧНО: Видаліть локальні файли private_key.pem та public_key.pem зі свого комп'ютера. Клієнт та Сервер будуть підтягувати їх з хмари автоматично.

5. Запуск Сервера
На машині отримувача запустіть серверний скрипт обраного модуля:
python server_basic.py

Ви маєте побачити повідомлення [+] УСПІХ: Приватний ключ завантажено... та [*] Сервер запущено....

6. Запуск Клієнта
На машині відправника запустіть клієнтський скрипт:
python client_basic.py

Клієнт автоматично створить папку, вказану в TARGET_FOLDER (за замовчуванням target_folder), і почне її моніторити.

7. Ексфільтрація
Створіть будь-який текстовий файл (наприклад, passwords.txt) з тестовими даними.
Помістіть цей файл у папку target_folder на машині Клієнта.

Спостерігайте за магією:
Клієнт миттєво підхопить файл, зашифрує його (RSA+AES), розіб'є на фрагменти (Chunks) і почне приховано відправляти через обраний хмарний сервіс із Jitter-паузами.
Сервер отримає всі частини, перевірить SHA-256 хеш, розшифрує дані та збереже оригінальний файл у папку stolen_data.
Сервер надішле клієнту маркер підтвердження (ACK), після чого клієнт безслідно видалить оригінальний файл зі своєї машини.
